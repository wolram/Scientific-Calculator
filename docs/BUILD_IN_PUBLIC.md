# Build in Public Playbook

## Objetivo

Transformar desenvolvimento em conteudo recorrente que gera confianca, distribuicao e vendas.

## Cadencia minima

- 3 posts por semana (antes/depois de features)
- 1 demo curta por semana (30-60s)
- 1 resumo quinzenal de aprendizados

## Formato de update

1. Problema que estava bloqueando
2. Solucao tecnica implementada
3. Resultado visual (gif/video/screenshot)
4. Proximo passo claro

## Ideias de conteudo

- Comparativo de UX: Scientific vs Financial vs Graphing
- Bastidores: parser de expressoes e precisao numerica
- Diferencas reais de desenvolvimento: SwiftUI x Compose x Swing
- Otimizacoes de performance por plataforma

## KPIs simples

- Crescimento de lista de espera
- Taxa de clique para pagina do produto
- Conversao por plataforma
- Retencao semanal de usuarios ativos
